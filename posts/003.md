# HOW I UPDATE THIS BLOG

As I was building this tiny site it was a priority for me to make it so that I could update it using only my belover *LINUX TERMINAL*.

First thing I did was that I make a little shell-script that scans the **/posts/** folder for entries. If it finds a new entry, it adds it to a .json-file which lists all entries. This was pretty straigh forward - basically just one like of code. It took a while to get it right though, but trial and error is the best (when it works in the end).

Then, whenever the site loads, a tiny little JavaScript function runs that reads all the posts and renders them in HTML.

This works really great for me. When I want to make a new post, I just:

* Create a new Git branch
* Write and save a new post in /posts/ (in markdown)
* Run the script
* Stage my new post
* Finally, commit and push that good stuff

I have no idea if this is a very common way of doing it, but I really like it. It's neat when you come up with solutions that works for you, you know?

I never bothered with markdown before, but it was really easy to get into.
